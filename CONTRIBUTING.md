# راهنمای مشارکت

&#x202b; برای تقویت فهرست پروژه‌های ضدجعل و ضدفرا‌د، لطفاً دستورالعمل‌های زیر را رعایت کنید تا افزوده‌ها قابل اتکا و یک‌دست بمانند.

## مراحل ارسال تغییرات
1. ریپو را **Fork** و برای هر تغییر شاخه جداگانه ایجاد کنید.
2. پروژه‌های جدید را در جدول دسته‌بندی مرتبط در `README.md` اضافه یا به‌روزرسانی کنید (حروف الفبا و توضیح کوتاه، فنی و بی‌طرف).
3. همان پروژه را با فیلدهای کامل در `projects.json` ثبت کنید: `name`, `github`, `url`, `category`, `description`, `tags`, `license` و در صورت امکان `tech`.
4. توضیح باید مختصر (کمتر از ~120 کاراکتر) و دقیق درباره نقش ضدجعل/ضدفرا‌د باشد؛ از لحن تبلیغاتی پرهیز کنید.
5. قبل از ارسال PR، اسکریپت زیر را اجرا کنید تا خطاهای ساختاری و لینک‌ها مشخص شوند:
   ```bash
   python scripts/validate_projects.py --check-urls --timeout 5.0
   ```

## معیارهای پذیرش
- پروژه باید **اوپن‌سورس و عمومی** باشد و مستقیماً به اصالت‌سنجی، احراز هویت، زنجیره تأمین یا تشخیص رسانه جعلی کمک کند.
- لینک GitHub/گیت‌های عمومی باید فعال و تست‌شده باشد؛ از مخازن خصوصی یا آرشیو شده استفاده نکنید.
- دسته‌بندی باید یکی از مقدارهای تعریف‌شده در [docs/SCHEMA.md](docs/SCHEMA.md) باشد و `license` خالی نماند (در صورت شک `UNKNOWN`).

## قوانین قالب‌بندی
- در `README.md` فقط از فرمت جدول هر بخش استفاده کنید و ترتیب نام پروژه‌ها را حروف الفبا نگه دارید.
- در `projects.json`، مقدار `category` باید با دسته‌بندی‌های README یکسان باشد و `tags` حتماً آرایه باشد.
- برای جزئیات فیلدها و مثال‌ها به [docs/SCHEMA.md](docs/SCHEMA.md) و [docs/USE-CASES.md](docs/USE-CASES.md) رجوع کنید.

## گزارش مشکل یا لینک خراب
- از قالب Issue «گزارش لینک خراب یا داده نادرست» استفاده کنید و URL/نام پروژه را درج کنید.
- اگر پروژه‌ای حذف شده یا لایسنس نامشخص است، در Issue توضیح دهید تا اصلاح یا جایگزینی انجام شود.
