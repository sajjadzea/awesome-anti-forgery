# نقشه مهارت و مسیر یادگیری برای Trust Systems و ضدجعل

## مهارت‌های فنی پایه
- مبانی AI/ML: طبقه‌بندی، تشخیص ناهنجاری، آشنایی با مدل‌های مولد و محدودیت‌هایشان.
- رمزنگاری پایه: امضای دیجیتال، هش، مدیریت کلید و پروتکل‌های SSL/TLS.
- مفاهیم بلاک‌چین و گراف: مرکل‌تری، Provenance، گراف دانش و کاربرد آن در زنجیره تأمین و هویت.
- امنیت نرم‌افزار و DevSecOps: SBOM، امضای آرتیفکت، CI/CD امن و اسکن آسیب‌پذیری.

## مهارت‌های غیر فنی
- تفکر سیستمی و مدل‌سازی مسئله اعتماد و جعل در هر صنعت.
- مذاکره و تسهیلگری با ذینفعان برای تعریف سیاست‌ها و سطح سخت‌گیری.
- نویسندگی و گزارش‌نویسی فنی برای ترجمه خروجی مدل به تصمیم تجاری.
- سواد رسانه‌ای و اخلاق داده برای تشخیص بایاس و اثرات اجتماعی.

## استفاده از این ریپو به‌عنوان سیلابس پروژه‌محور
1. **پروژه ۱: ساخت سامانه تأیید سند**
   - ابزار: Blockcerts، OpenAttestation، کتابخانه‌های رمزنگاری.
   - خروجی: صدور گواهی، API راستی‌آزمایی، داشبورد وضعیت امضا.
2. **پروژه ۲: ساخت سامانه تشخیص رسانه جعلی**
   - ابزار: FaceForensics++، DeepfakeDetector/DeepSafe، داشبورد Streamlit.
   - خروجی: امتیاز ریسک، گزارش فورنزیک، آستانه تصمیم‌گیری برای انتشار.
3. **پروژه ۳: ساخت داشبورد ریسک زنجیره تأمین**
   - ابزار: Hyperledger، OriginTrail، Rule Engine ساده، داشبورد React.
   - خروجی: نقشه Provenance، هشدار تقلب، گزارش ممیزی برای حسابرسی.

## گام‌های پیشنهادی برای یادگیری
- یک دسته‌بندی را انتخاب کنید (سند، هویت، زنجیره تأمین یا رسانه) و سه پروژه از `projects.json` را Clone و تست کنید.
- KPI و سنجه‌های خطا را تعریف کنید و خروجی پروژه‌ها را با داده کوچک آزمایش کنید.
- Playbook کوتاه بنویسید که چطور یک تیم محصول می‌تواند این استک را در محیط واقعی مستقر کند.
- یافته‌ها را در قالب PR به docs اضافه کنید تا جامعه بتواند نقد و بهبود دهد.
